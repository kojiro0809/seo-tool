# SEO Analyzer - 記事構成・見出し抽出ツール (Python / Streamlit)

指定したURLのWebページから「タイトル」「文字数」「見出し構成（H1〜H3）」を瞬時に抽出し、競合サイトのSEO記事の論理構造を丸裸にする分析ツールです。
🌐 **[公開中のWebアプリはこちらから体験できます](https://seo-tool-ex8mfvjkmaz294afbtqbvl.streamlit.app/)**

## 🎯 開発の背景とビジネス課題の解決
本アプリは、大学の授業で得た知識が「実社会の課題解決」に直結した経験から生まれました。

私は大学の専門科目で「HTMLのDOMツリーやタグの論理構造」について深く学ぶ一方、並行して受講していた一般教養の授業で「デジタルマーケティングにおけるSEOの重要性」について触れる機会がありました。その中で、「検索エンジンに評価されるには、H1やH2といった見出しの階層構造が極めて重要である」という事実を知りました。

ここで私の中に一つの疑問が浮かびました。
**「世の中のWebライターやマーケターは、競合サイトの構造を調べるために、毎回ブラウザの開発者ツールを開いてノイズ（CSSや広告）だらけのHTMLから手作業で見出しを拾い集めているのだろうか？それはあまりにも非効率な『業務の苦痛』ではないか」**

そこで、「自分が学んだHTML解析の知識を使えば、この泥臭い手作業を『1記事5分から1秒』に短縮できる」と思いつき、非エンジニアでも直感的に使える業務ハックツールとして本アプリを開発しました。
**「見た目の装飾をすべて削ぎ落とし、HTMLの論理構造（DOMツリー）だけを抽出できれば、この作業を1記事5分から『1秒』に短縮できる」**

そう考え、BeautifulSoupを用いたWebスクレイピング技術と、直感的なUIを提供するStreamlitを組み合わせ、非エンジニアでも直感的に使える業務ハックツールとして本アプリを開発しました。

## 🛠 技術的な工夫と実務への適応（こだわり）
単なる「HTMLの取得プログラム」で終わらせず、実際のマーケティング業務で使える水準に近づけるため、以下の工夫を実装しています。

### 1. 公開アプリケーションとしてのセキュリティと堅牢性（SSRF・OOM対策）
任意のURLを受け付けるスクレイピングツールの性質上、重大な脆弱性になり得るリスクへの対策を講じています。
- **SSRF対策:** ローカルネットワークや内部ファイルへの不正アクセスを防ぐため、プロトコルを `http://` および `https://` に厳格に制限。
- **OOM対策:** 巨大なファイル（動画や重いPDFなど）のURLを入力された際のサーバーメモリ枯渇（Out Of Memory）を防ぐため、リクエストをストリーム化し、Content-Lengthが5MBを超える場合は処理を強制中断するフェイルセーフを実装。

### 2. 「生のWebデータ」の泥臭い処理（文字化け対策）
日本のWebサイトは文字コード（UTF-8, Shift-JIS, EUC-JP等）が統一されていません。`response.apparent_encoding` を用いてレスポンスの文字コードを動的に判定・適用することで、日本語特有の文字化け（Mojibake）を防止し、堅牢性を高めています。

### 3. User-Agentの最適化によるアクセシビリティ向上
標準の `requests` のアクセスではBotとして弾かれるセキュリティ（WAF等）が多いため、iOS（iPhone）のUser-Agentヘッダーを付与しています。これにより、モバイルフレンドリーな構成の取得と、アクセス拒否の回避を両立しています。

### 4. 分析フローへのシームレスな結合（CSV出力）
画面上に結果を表示するだけでなく、マーケターがスプレッドシートやExcelに貼り付けて競合比較表を作る業務フローを想定し、抽出したDOM構造を `pandas` で整形し、ワンクリックでCSVダウンロードできる機能を実装しました。

## 🌱今後の課題（SPAへの対応）
本ツールは `requests` による静的HTMLの取得を前提としています。そのため、ReactやVue.jsなどで構築され、JavaScriptでクライアントサイドレンダリングされるモダンなSPA（Single Page Application）のサイトでは、見出しが取得できない課題があります。
今後は、`Selenium` や `Playwright` などのヘッドレスブラウザを導入し、動的レンダリング後のDOMツリーを取得できるようアーキテクチャのアップデートを検討しています。
